# T207 - Studio Console 정보구조/비주얼 시스템 설계

## Spec

### 문제

- 현재 UI는 기능은 갖췄지만 “작업 콘솔”로서 정보 위계와 액션 우선순위가 약하다.
- 리스트/실시간/설정 화면의 시각 언어가 분절되어 일관된 브랜드 경험이 부족하다.

### 목표

- `Studio Console` 콘셉트로 정보 구조와 시각 시스템을 재정렬한다.
- 데스크톱/모바일 모두에서 “현재 상태 + 다음 액션”이 즉시 보이는 UI를 만든다.

### 범위

- 포함:
  - 전체 IA(상단 바, 페이지 헤더, 주 작업 패널, 보조 패널) 정의
  - 디자인 토큰(타이포 스케일, 색상 역할, 간격, radius, elevation) 정리
  - 핵심 화면(리스트/실시간/설정) 레이아웃 스펙 정의
  - 적용 우선순위와 마이그레이션 계획 수립
- 제외:
  - 한 번에 전체 화면 전면 교체

### 해결방안

- 콘솔 레이아웃을 2-tier로 통일한다.
  - Tier 1: 페이지 목적/상태 헤더
  - Tier 2: 작업 패널(Primary) + 보조 컨텍스트(Secondary)
- 디자인 토큰을 theme 단일 소스로 정리해 화면 간 편차를 줄인다.
- 모바일은 단일 컬럼 + sticky action strip로 재해석해 기능 손실 없이 단순화한다.
